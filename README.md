# Chastushki Research Project

Этот каталог содержит исследование и продуктовые артефакты по идее сервиса генерации русских частушек.

## Что здесь лежит

### 1. Быстрый вход

- `README.md`
  - этот файл
  - объясняет состав документов и рекомендуемый порядок чтения

- `summary.md`
  - короткая сводка по идее
  - даёт общий вердикт, почему идея выглядит жизнеспособной и как её лучше позиционировать

- `founder_memo.md`
  - короткий стратегический документ на 1-2 страницы
  - полезен, если нужно быстро понять суть идеи, opportunity, wedge, риски и что валидировать первым

## Исследование

- `market_scan.md`
  - обзор аналогов, конкурентов, смежных продуктов и академических/opensource решений
  - отвечает на вопрос: есть ли похожие проекты и насколько занята ниша

- `technical_plan.md`
  - техническое проектирование сервиса без реализации
  - отвечает на вопрос: как это можно построить, какие есть варианты pipeline, что брать в MVP, какие есть риски по качеству, правам и safety

## Продуктовое проектирование

- `design_doc.md`
  - основной продуктовый design-doc
  - содержит позиционирование, аудитории, JTBD, MVP scope, user flows, IA, UX-принципы, монетизацию, GTM-гипотезы и риски

- `product_strategy.md`
  - более стратегическая часть design-doc
  - фокус на positioning, ICP, JTBD, wedge, differentiation и запуск

- `product_experience.md`
  - более UX-ориентированная часть design-doc
  - фокус на user flows, экранной структуре, форматах выдачи, loop'ах улучшения, монетизации и GTM

## PRD для MVP

- `prd_mvp.md`
  - главный PRD-документ для MVP
  - содержит цель MVP, scope, user stories, требования, acceptance criteria, метрики, release criteria, rollout assumptions и открытые вопросы

- `prd_mvp_core.md`
  - черновая базовая часть PRD
  - полезна, если нужно смотреть PRD по слоям

- `prd_mvp_metrics.md`
  - дополняющая часть PRD
  - содержит success metrics, activation, guardrails, release criteria и monetization assumptions

## Founder memo и вспомогательные материалы

- `founder_memo.md`
  - финальная версия founder memo
  - лучший краткий документ для обсуждения идеи как стартап-гипотезы

- `founder_memo_draft.md`
  - исходный черновик founder memo
  - можно читать только если хочется видеть промежуточную формулировку

- `founder_memo_notes.md`
  - sharpening notes к founder memo
  - содержит честные замечания про moat, скепсис, ограничения и что валидировать в первую очередь

## Рекомендуемый порядок чтения

### Вариант 1. Если нужно быстро понять идею за 10-15 минут

Читай так:
1. `summary.md`
2. `founder_memo.md`
3. `prd_mvp.md`

Этого достаточно, чтобы понять:
- стоит ли идея внимания
- как она позиционируется
- каким должен быть MVP

### Вариант 2. Если нужно полное понимание продукта

Читай так:
1. `summary.md`
2. `market_scan.md`
3. `technical_plan.md`
4. `design_doc.md`
5. `prd_mvp.md`
6. `founder_memo.md`

Это даёт полный путь:
- рынок
- техподход
- продуктовая логика
- MVP
- краткая стратегическая упаковка

### Вариант 3. Если нужно обсуждать идею как бизнес/стартап

Читай так:
1. `founder_memo.md`
2. `summary.md`
3. `market_scan.md`
4. `prd_mvp.md`
5. `founder_memo_notes.md`

Это полезно, если нужно:
- обсуждать потенциал идеи
- смотреть на риски
- оценивать moat и что валидировать первым

### Вариант 4. Если нужно работать именно над MVP

Читай так:
1. `summary.md`
2. `design_doc.md`
3. `prd_mvp.md`
4. при необходимости `technical_plan.md`

Это лучший порядок для перехода к следующему этапу проектирования.

## Что считать основными документами

Если смотреть только на главный комплект, то это:
- `summary.md`
- `market_scan.md`
- `technical_plan.md`
- `design_doc.md`
- `prd_mvp.md`
- `founder_memo.md`

Остальные файлы, это либо составные части, либо вспомогательные материалы.

## Текущий статус проекта

На текущем этапе проект находится в состоянии:
- исследование выполнено
- продуктовая рамка собрана
- MVP описан
- founder memo собран
- реализации ещё нет

Следующие естественные шаги:
1. `validation plan`
2. `landing page spec`
3. при необходимости более подробный go-to-market или testing plan
